You are helping a developer set up the `test-video-hook-poc` example app inside the catalyst-core repository.

Your goal is to generate three gitignored config files that must exist locally before the app can build:
  - config/config.json
  - google-services.json
  - GoogleService-Info.plist

---

## Step 1 — Discover the environment using shell commands

Run these commands to collect environment info automatically. Do not ask the user for things you can discover yourself.

```bash
# Local machine IP (LAN address for emulator/simulator to reach dev server)
ipconfig getifaddr en0 || ipconfig getifaddr en1

# Android SDK path candidates
ls ~/Android/ 2>/dev/null && echo "found: ~/Android/" || echo "not found"
ls ~/Library/Android/sdk 2>/dev/null && echo "found: ~/Library/Android/sdk" || echo "not found"

# Available Android emulators
emulator -list-avds 2>/dev/null || echo "emulator not in PATH"

# Available iOS simulators
xcrun simctl list devices available 2>/dev/null | grep -E "iPhone|iPad" | head -10

# Check if Firebase files already exist
ls config/config.json google-services.json GoogleService-Info.plist 2>/dev/null
```

---

## Step 2 — Ask only what you could not discover

After running the above, ask the user only for values you couldn't determine:
- Android package name (e.g. `com.yourname.videohookpoc`) — cannot be auto-detected
- iOS bundle ID (e.g. `com.yourname.videohookpoc`) — cannot be auto-detected
- App display name (e.g. "Video Hook POC") — cannot be auto-detected
- Whether they have a real Firebase project (yes / skip)
  - If yes: ask for the values from their existing `google-services.json` and `GoogleService-Info.plist` from Firebase Console
